# notes/

Scratch space for working notes that don't belong in `docs/` — feedback reports, draft implementation plans, session journals, anything load-bearing during a piece of work but not part of the user-facing documentation.

**This folder is gitignored except for this README.** Drop whatever you want here; nothing else is tracked. If something graduates into a real artifact, move it to:

- `docs/<name>.md` — if it becomes user-facing reference or guide
- `CHANGELOG.md` — if it captures a behavior change
- A commit message — if it's the *why* behind a diff

Anything left here long-term is fair to delete. Don't link to `notes/` from `docs/` or `README.md` — the path is contributor-local, not a stable URL.
